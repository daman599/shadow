# Contributing to Shadow

First of all, thank you for considering contributing to **Shadow** ✨  
Shadow is a collection of beautiful box, text, and drop shadows that developers can copy and use instantly.

We welcome all kinds of contributions — from bug fixes to new shadow styles.

---

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork

   git clone https://github.com/your-username/shadow.git

3. Install dependencies

   npm install

4. Start the development server

   npm run dev

---

## 🌿 Branching Rules

- Do NOT push directly to `main`
- Create a new branch:

  feature/shadow-name  
  fix/bug-description  
  docs/update-readme  

Example:

  git checkout -b feature/soft-neon-shadow

---

## 🎨 Adding a New Shadow

When adding a new shadow:

- Give it a clear and meaningful name
- Add proper `id`
- Include:
  - CSS value
  - Tailwind class (if applicable)
  - Category (box, text, drop)

Keep it:
- Clean
- Reusable
- Production-ready

---

## 🧹 Code Style

- Follow existing file structure
- Keep components reusable
- Avoid unnecessary complexity
- Make sure formatting is consistent

If linting is enabled, run:

  npm run lint

---

## 📸 UI Changes

If your contribution affects UI:

- Add a screenshot in your Pull Request
- Explain what was improved or changed

---

## 🐛 Reporting Bugs

When reporting a bug, please include:

- What happened
- Expected behavior
- Steps to reproduce
- Screenshot (if possible)

---

## 📦 Pull Request Guidelines

Before submitting a PR:

- Make sure the project builds successfully
- Check for console errors
- Keep PR focused (one feature per PR)

---
